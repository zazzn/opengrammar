Ollama — Which model for my GPU?

Reference for the local (Ollama) grammar/tone model. Sizes are the Ollama default Q4 quant. The extension reads your installed models live; use this to decide what to ollama pull and pick. Picks favour fast, strong instruction-following models (qwen2.5 family); reasoning models (qwen3) give better edits but are slower.

VRAM headroom: leave ~1–2 GB free. A model that doesn't fit spills to system RAM and gets very slow. Skip coder/embed/vision variants — they're poor at prose grammar.

Gaming / GPU-heavy tasks: Ollama keeps the model in VRAM for the idle period after each use. The extension can't detect a game launching (it's a browser extension), so to free VRAM: set “Unload model when idle” (in the popup) to Immediately or 30 s, and/or click “Unload now” in the popup before you start a game. This is exactly why running Ollama on a separate machine avoids the contention entirely.

Debug & Tuning

Capture what OGrammar produces (inline fixes, corrections, tone rewrites, autocomplete) so you can share a compact log for tuning. Off by default; no overhead when off. Nothing is sent anywhere — the log stays on this device until you copy it.
